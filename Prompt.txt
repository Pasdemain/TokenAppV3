# TokenApp Recreation Prompt for AI

## Project Overview
Create a Flask web application called "TokenApp" that manages favor tokens between users and includes a collaborative shopping list feature. The app should be deployable on Railway with PostgreSQL database.

## Core Features Required

### 1. User Authentication System
- User registration and login with username/password
- Password hashing with bcrypt
- Session management
- Simple profile page

### 2. Token Management System
- **Create Tokens**: Users can create favor tokens for other users with:
  - Recipient selection (dropdown of users)
  - Token name (e.g., "Help with moving")
  - Description 
  - Duration in minutes
  - Status tracking (available, in_progress, completed)
- **Dashboard**: Show user's tokens (received and created)
- **Token Usage**: Recipients can mark tokens as "in progress" and "completed"

### 3. Shopping Lists Feature
- **Create Lists**: Users can create shopping lists with optional sharing
- **Add Items**: Add items with name, quantity, and unit (kg, pcs, L, etc.)
- **Collaborative**: Multiple users can add/complete items on shared lists
- **Progress Tracking**: Visual progress indicators
- **Delete Lists**: Only list creators can delete lists
- **Toggle Items**: Mark items as complete/incomplete

### 4. Admin Panel
- Password-protected admin panel at `/admin`
- Admin password: `Tom123`
- Ability to clear all data (users, tokens, shopping lists)

## Technical Requirements

### Database Schema (PostgreSQL)
```sql
-- Users table
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    username VARCHAR(20) UNIQUE NOT NULL,
    password_hash TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Tokens table
CREATE TABLE tokens (
    id SERIAL PRIMARY KEY,
    creator_id INTEGER REFERENCES users(id),
    recipient_id INTEGER REFERENCES users(id),
    name VARCHAR(50) NOT NULL,
    description TEXT,
    duration_minutes INTEGER NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    used_at TIMESTAMP,
    status VARCHAR(20) DEFAULT 'available' CHECK (status IN ('available', 'in_progress', 'completed'))
);

-- Shopping lists table
CREATE TABLE shopping_lists (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    created_by INTEGER REFERENCES users(id),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    is_active BOOLEAN DEFAULT TRUE
);

-- Shopping items table
CREATE TABLE shopping_items (
    id SERIAL PRIMARY KEY,
    list_id INTEGER REFERENCES shopping_lists(id) ON DELETE CASCADE,
    name VARCHAR(100) NOT NULL,
    quantity VARCHAR(20) DEFAULT '1',
    category VARCHAR(50) DEFAULT 'pcs',
    is_completed BOOLEAN DEFAULT FALSE,
    added_by INTEGER REFERENCES users(id),
    completed_by INTEGER REFERENCES users(id),
    added_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    completed_at TIMESTAMP
);

-- Shopping list members table (for sharing)
CREATE TABLE shopping_list_members (
    id SERIAL PRIMARY KEY,
    list_id INTEGER REFERENCES shopping_lists(id) ON DELETE CASCADE,
    user_id INTEGER REFERENCES users(id),
    role VARCHAR(10) DEFAULT 'member' CHECK (role IN ('owner', 'member')),
    joined_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(list_id, user_id)
);
```
